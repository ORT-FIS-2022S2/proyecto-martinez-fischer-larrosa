## Técnicas aplicadas y aprendizajes

En cuanto a las tecnicas y aprendizajes que decidimos aplicar para la creacion y funcionamiento del proyecto tuvimos una gran variedad, las cuales fueron:

- GitHub: Al igual que la primer parte del proyecto, utilizamos GitHub para realizar la segunda. Nos fue util el uso de este programa ya que a la hora de dividirnos las tareas se nos hacia facil el uso de diferente ramas para no sobreponer el trabajo del otro y que todo estuviera bien organizado. Hicimos uso de varias ramas, por un lado una para los servicios, en la cual se iba a codificar toda la parte de dominio, luego la rama interfaz utilizada para el frontend del proyecto, asi como tambien una rama integracion en la cual integramos toda la parte del frontend con el backend, lo cual fue lo que mas nos costo. Por ultimo creamos la rama documentacion para documentar todo lo que habiamos hecho y volvimos a usar la rama develop para integrar todo antes de mandarlo al main. Ademas consideramos que fue de gran importancia utilizar GitHub para este proyecto ya que algunos de los integrantes no conocian este mundo y era nescesario que se acostumbraran a como funcionan los commits, branches, markdown, issues y todo lo que formo parte dentro de GitHub para el proyecto.

- Node JS: Para poder codificar codigo JavaScript fuera del navegador web hicimos uso de Node JS, el cual sirve para ejecutar JavaScript desde nuestro ordenador sin ningun browser. Hicimos uso del npm para instalar librerias externas en el package.json lo cual fue fundamental a la hora de trabajar en el proyecto. Al ser una nueva herramienta de trabajo para un integrante del equipo, fue fundamental hacer uso del mismo para que vaya aprendiendo como funciona el mismo y la instalacion de paquetes.

- Material Design: Dentro de todas las tecnicas implementadas, una de las mas complejas fue Material Design. Al principio no nos salia nada, pero luego de varios intentos y mucha investigacion logramos poder hacer uso correcto de diferentes elementos del mismo. Es una buena pratica ya que hace que el frontend de la aplicacion se vea mejor para los usuarios, lo cual es algo que seguramente utilizemos en nuestra carrera.

- SQLite: Fuera de lo que fue propuesto por el curso, una de las tecnicas que aplicamos fue SQLite, el cual es un programa que nos permite crear una base de datos de forma sencilla para poder guardar toda la informacion de los usuarios y los partidos de forma persistente.

- API: Decidimos hacer uso de una API en nuestro proyecto para los servicios que implementamos. Uno de nuestros integrantes no conocia lo que era, por lo que al principio estaba medio confundido con respecto a lo que era la API, pero por suerte se fue adaptando y aprendiendo sobre lo que era la misma.

- Prisma: Por otro lado, hicimos uso de Prisma para poder mergear la API con la base de datos, lo cual nos fue de gran utilidad. Ademas, estuvo bueno aplicar esto junto a la API y a SQLite ya que Mateo no conocia estos y fue importante para el saber sobre ellos y como funcionan.

- ESLint: Hicimos uso de ESLint, como se pedia en la propuesta para poder verificar los estandares de codificacion. Es algo que no conociamos y fue util a la hora de trabajar en el proyecto.

- Jest: Para realizar las pruebas unitarias hicimos uso de Jest, una tecnica que dimos en el curso la cual fue bueno aplicar para ir conociendo y metiendonos en el mundo del testing unitario.

- Tesing exploratorio: Al final del proyecto nos toco hacer testing exploratorio sobre otro proyecto. Aplicamos tecnicas de caja negra con la tecnica de particion equivalente en diferentes casos de prueba. Estuvo bueno ir conociendo como funciona la etapa final de un proyecto, ya que el testing es fundamental para esta etapa. Ademas hicimos el uso de las issues en GitHub a la hora de el testing exploratorio.

### Presentacion NODE JS

Nosotros tuvimos que crear la presentacion de NodeJS, el cual fue utilizado en el proyecto.

NodeJS es un entorno de ejecucion de JavaScript multiplataforma, de codigo abierto, asincronico y orientado a eventos. Los programas escritos en JS son compilados, optimizados e interpretados por la maquina virtual V8 de chrome.

Sirve para ejecutar JavaScript desde nuestro ordenador sin ningun browser. Es ideal para construir aplicaciones altamente escalables y de uso intensivo de operaciones de entrada y salida o para crear sitios web dinamicos muy eficientes escritos en JavaScript.
